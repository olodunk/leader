
# 中层干部民主测评配置映射关系说明 (Mapping Documentation)

本文档详细说明了系统中**被考核人 (Examinee)** 与 **考核人 (Rater)** 身分识别的逻辑映射关系。

## 1. 被考核人与中层管理人员表映射 (Examinee Mapping)

系统根据人员信息表中的 **“员工角色”** 和 **“部门名称”** 两个字段，将其自动归类到测评配置表中的某一“列” (Column)。

| # | 测评配置中的被考核人角色 (Target Column) | 对应人员表匹配条件 (Source Condition from Personnel Table) | 备注 |
| :--- | :--- | :--- | :--- |
| 1 | **院长助理** | 角色 = `院长助理` | 直接匹配 |
| 2 | **职能部门正职** | 角色 = `职能部门正职` | 直接匹配 |
| 3 | **职能部门副职** | 角色 = `职能部门副职` | 直接匹配 |
| 4 | **研究所正职** | 角色 = `研究所正职` | 直接匹配 |
| 5 | **研究所副职** | 角色 = `研究所副职` | 直接匹配 |
| 6 | **两中心正职** | 角色包含 `中心` 且包含 `正职` <br> **且** 部门 IN [`兰州化工研究中心`, `大庆化工研究中心`] | 例如：中心正职 |
| 7 | **两中心副职** | 角色包含 `中心` 且包含 `副职` <br> **且** 部门 IN [`兰州化工研究中心`, `大庆化工研究中心`] | |
| 8 | **昆冈班子副职 (北京)** | 角色包含 `中心` 且包含 `副职` <br> **且** 部门 = `昆冈先进制造（北京）有限公司` | (注意部门名称精确匹配) |
| 9 | **所属分公司 (兰州、抚顺) 班子正职** | 角色包含 `中心` 且包含 `正职` <br> **且** 部门 IN [`昆冈兰州分公司`, `昆冈抚顺分公司`] | |
| 10 | **所属分公司 (兰州、抚顺) 班子副职** | 角色包含 `中心` 且包含 `副职` <br> **且** 部门 IN [`昆冈兰州分公司`, `昆冈抚顺分公司`] | |
| - | *(已排除)* | 角色包含 `基层` | 基层人员不作为被考核对象 |

## 2. 考核人与账号映射 (Rater Mapping)

系统根据登录账号所属的 **“部门类型”**、**“部门名称”** 以及 **“账号类型”** (后缀 L/P/D/C/E)，赋予其对应的打分角色权限 (Row)。

> **账号类型代码说明**：
> *   **L**: 院领导 (Leader)
> *   **P**: 正职 (Principal)
> *   **D**: 副职 (Deputy)
> *   **C**: 中心基层领导 (Center/Grassroots)
> *   **E**: 其他员工 (Employee)
